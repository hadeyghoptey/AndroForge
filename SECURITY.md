# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 2.x     | ✅ Yes |
| < 2.0   | ❌ No |

## Reporting a Vulnerability

If you discover a security vulnerability in AndroForge, please report it by opening an issue on the [GitHub repository](https://github.com/hadeyghoptey/AndroForge/issues).

Do **not** open a public issue for critical vulnerabilities. Instead, email the maintainer directly at hadamanash2023@gmail.com.

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (if applicable)

You should receive a response within 48 hours. If the issue is confirmed, a patch will be released as soon as possible.

## Responsible Disclosure

We kindly ask that you:
- Allow time for a fix before publicly disclosing
- Do not exploit the vulnerability beyond proving its existence
- Follow all applicable laws

## Scope

This security policy applies to the AndroForge codebase only. The tool is designed for authorized testing — misuse is not the responsibility of the maintainers.
